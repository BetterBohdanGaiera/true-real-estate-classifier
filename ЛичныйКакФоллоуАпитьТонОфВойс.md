# Личный Tone of Voice для Follow-up сообщений

## Общий стиль

- Неформальный, но не панибратский
- Без "йо", "чел", сленга
- Структурированный, но не сухой
- Информативный без воды

## Форматирование

- **Жирный** для заголовков разделов
- Короткие предложения, каждое с новой строки
- Без точек в конце (чтобы не выглядело как LLM)
- Списки через дефис для деталей
- Нумерация для последовательных пунктов (1. 2. 3.)

## Структура сообщения

```
Приветствие (короткое)

Контекст одной фразой

**Что сделано:**

**Пункт 1**
Детали
Ещё детали

**Пункт 2**
...

**Над чем работаю / Что дальше:**

1. **Задача**
Описание

2. **Задача**
...
```

## Правила

1. Не начинать с "Итак", "Ну", "Короче"
2. Не использовать эмодзи (если не просят)
3. Длинные предложения разбивать на несколько коротких
4. Технические термины оставлять как есть (Zoom, CRM, API)
5. Названия документов/файлов в кавычках
6. Показывать глубину работы через конкретику, не через слова "много", "глубоко"

## Пример

```
Всем привет!

Апдейт по проекту

**Что сделано:**

**Название фичи**
Что конкретно сделано
Как это работает

**Другая фича**
Детали
- Подпункт если нужен
- Ещё подпункт

**Над чем работаю:**

1. **Задача**
Короткое описание статуса

2. **Задача**
Что именно делается
```

## Чего избегать

- Точки в конце каждого предложения
- Слишком формальные обороты ("Хотел бы сообщить", "Позвольте доложить")
- Слишком неформальные ("Короч", "Чуваки")
- Пустые фразы ("Проделана большая работа", "Всё идёт по плану")
- Эмодзи без явного запроса
