## 3. Ключевые метрики из транскрипта

### 3.1 RevPAR (Revenue Per Available Room)

**Формула:** RevPAR = (Занятость × Средний чек) / 100

**Примеры расчета:**
- Юнит стоимостью $80,000 с RevPAR $14,000 = **17.5% валовая доходность**
- После вычета операционных расходов (20-40%) = **10-12% чистая доходность**
- Kempinski RevPAR: **$100,000/год** (высокий показатель из-за больших юнитов и пентхаусов)

## 5. Функции софта EstateMarket.io

### 5.1 Карта и визуализация

- **Хитмапа (тепловая карта):** показывает насыщенность территории юнитами
  - Красные точки = высокая концентрация юнитов
  - Зеленые точки = низкая концентрация
- **Районы:** Чангу, Кута, Убуд, Семиньяк (выделяются синей рамкой)
- **Радиус анализа:** 1-2 км вокруг выбранного юнита

### 5.2 Статистика по радиусу (пример 1 км)

| Параметр | Значение |
|----------|----------|
| Занятость | 63% |
| Средний чек | $133 |
| RevPAR | $30,000/год |
| Количество юнитов | 214 |

### 5.3 Фильтры и сортировка

- По занятости (0-100%)
- По RevPAR
- По площади (м²)
- По цене за ночь (до $1,600/сутки для премиум)
- По доходности (%)

### 5.4 Дополнительные функции

- **Списки:** создание подборок конкурентов для сравнения
- **Экспорт:** данные в Excel
- **Поиск:** по названию проекта
- **История цен:** все прошлые даты бронирований

## 7. Ограничения и замечания

1. **Данные только с Booking.com** - Airbnb пока не подключен
2. **Большие отели** (Hyatt, Hilton, Kempinski) размещают не все номера на Booking
3. **Новые юниты** имеют минимум исторических данных
4. **Данные обновляются ежедневно** через парсинг Booking
5. **Прямые сделки** отелей не отображаются в системе
