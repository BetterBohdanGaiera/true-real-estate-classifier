# Estate Market: Софт для анализа рынка недвижимости Бали

## Краткое описание

Estate Market (EstateMarket.io) — это собственный IT-продукт компании True Real Estate, который сканирует Booking.com 24/7/365 для сбора реальных данных о заполняемости и ценах аренды недвижимости на Бали. Инструмент позволяет принимать инвестиционные решения на основе достоверных данных.

## Ключевые моменты

- Автоматический сбор данных с Booking.com каждый день
- Десятки тысяч объектов в базе данных
- Анализ заполняемости, цен и доходности по каждому юниту
- Возможность выбора любого временного периода
- Расчет ориентировочной рыночной стоимости объекта

---

## Как работает софт

### Принцип работы

1. Программа знает количество номеров/вилл в каждом отеле/комплексе
2. Каждый день робот проверяет доступность номеров на Booking.com
3. Вычисляется процент заполняемости: `занятые / всего`
4. Собирается цена за ночь для каждого типа юнита
5. Данные агрегируются за любой выбранный период

### Пример расчета

> В Munno Villa доступно 9 вилл. Сегодня свободно 5 → значит заняты 4.
> Заполняемость = 4/9 = 44%

## Интерфейс сайта

### Главная страница

Каждая карточка объекта содержит:

| Параметр | Описание |
|----------|----------|
| **Название** | Название отеля/комплекса |
| **Рейтинг** | Рейтинг на Booking.com (напр. 8.9) |
| **Тип** | Villa, Hotel, Resort и т.д. |
| **Местоположение** | Район (напр. Badung, Ubud) |
| **Units** | Количество юнитов в комплексе |
| **Types of units** | Количество типов номеров |
| **Average occupancy** | Средняя заполняемость за 30 дней |
| **Prices per night** | Диапазон цен за ночь |
| **Rental income** | Доход от аренды (расчетный) |

### Фильтры

- **Выбор города**: Ubud, Canggu, Seminyak и др.
- **Тип недвижимости**: Виллы, Отели, Апартаменты, Гостевые дома
- **Дополнительные параметры**: WiFi, Parking, Pool и т.д.
- **Ценовой диапазон**: Min/Max цена
- **Сортировка**: По заполняемости, цене, типу

> **Важно!** Всегда проверяйте и сбрасывайте фильтры перед поиском. Фильтры могут сохраняться после обновления страницы.

## Детальная карточка объекта

### Временные срезы

Можно выбрать любой период для анализа:
- Последние 30 дней (по умолчанию)
- Конкретные даты
- Помесячно для анализа сезонности

### Показатели по каждому типу юнита

| Показатель | Описание | Как считается |
|------------|----------|---------------|
| **Available days for rent** | Дней данных в выборке | Важно для валидации данных |
| **Average price** | Средняя цена за период | Сумма всех цен / количество дней |
| **Price range** | Разброс цен (min-max) | Показывает сезонные пики |
| **Occupancy** | Средняя заполняемость | % занятых дней за период |
| **Rental income** | Доход от аренды | Цена × Заполняемость × Дни |
| **Estimated Net Profit** | Чистая прибыль | Доход минус 40% расходов |
| **Estimated Price** | Рекомендуемая цена | Чистая прибыль × 10 лет |

### Формула расчета цены объекта

```
Estimated Price = Чистая прибыль за год × 10
```

Это стандартный мультипликатор для готовых объектов на рынке (10-летняя окупаемость).

## Практическое применение

### Для продажи

1. Найти аналогичный объект в том же районе
2. Показать клиенту реальные данные заполняемости
3. Сравнить с предлагаемым объектом:
   - "Этот старый отель зарабатывает X при такой заполняемости"
   - "Наш новый объект премиум-класса будет работать лучше"
   - "Расчетная рыночная цена аналога — $294,000, наш юнит стоит $115,000"

### Для анализа сезонности

- Выбирать разные месяцы
- Смотреть пиковые периоды (ноябрь-январь)
- Анализировать низкие сезоны (февраль)
- Понимать разброс цен в зависимости от загрузки

### Для оценки инвестиций

- Видеть реальную доходность аналогичных объектов
- Сравнивать разные районы и типы недвижимости
- Прогнозировать доходность новых проектов

## Особенности и ограничения

### На что обращать внимание

1. **Available days for rent** — проверяйте, что данные полные
   - Если выбрали 30 дней, а показано 5 — данные неполные
   - Новые номера могут иметь мало истории

2. **Сбои в данных** — редко, но бывают пропуски
   - Один потерянный день — не критично
   - Много пропусков — данные недостоверны

3. **Ограничения Booking.com**
   - Не все номера отображаются на Booking
   - Крупные отели (как Kempinski с 456 номерами) показывают только часть

### Фильтры типов

Фильтр "Тип недвижимости" не всегда корректен — данные берутся из Booking.com, который может неточно классифицировать объекты.

## Рекомендации по работе

1. **Регулярно изучать софт** — развивает насмотренность рынка
2. **Сравнивать месяцы** — понимать сезонность
3. **Готовить примеры заранее** — находить сильные кейсы для презентаций
4. **Проверять фильтры** — сбрасывать перед каждым новым поиском
5. **Валидировать данные** — проверять количество дней в выборке
