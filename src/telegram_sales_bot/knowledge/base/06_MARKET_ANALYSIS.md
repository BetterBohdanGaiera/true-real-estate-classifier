# Анализ рынка: Estate Market и интерпретация данных

> Как использовать софт и читать рыночные показатели

---

## Estate Market — обзор

### Что это

Собственный IT-продукт для анализа рынка недвижимости Бали.

### Как работает

1. Сканирует Booking.com **24/7/365**
2. Знает количество юнитов в каждом комплексе
3. Ежедневно проверяет доступность
4. Вычисляет заполняемость и цены
5. Агрегирует данные за любой период

### Пример принципа расчета

```
В комплексе 9 вилл. Сегодня свободно 5.
Заняты: 9 - 5 = 4
Заполняемость: 4 / 9 = 44%
```

## Интерфейс Estate Market

### Карточка объекта содержит

| Параметр | Описание |
|----------|----------|
| Название | Название отеля/комплекса |
| Рейтинг | Рейтинг на Booking.com |
| Тип | Villa, Hotel, Resort |
| Местоположение | Район |
| Units | Количество юнитов |
| Average occupancy | Средняя заполняемость (30 дней) |
| Prices per night | Диапазон цен |
| Rental income | Расчетный доход |

### Фильтры

- Город (Ubud, Canggu, Seminyak)
- Тип (Виллы, Отели, Апартаменты)
- Цена (Min/Max)
- Сортировка (по заполняемости, цене)

> **Важно:** Всегда сбрасывать фильтры перед новым поиском!

## Детальные показатели объекта

| Показатель | Описание | Как считается |
|------------|----------|---------------|
| **Available days** | Дней данных в выборке | Для валидации |
| **Average price** | Средняя цена | Сумма / дни |
| **Price range** | Разброс min-max | Сезонные пики |
| **Occupancy** | Заполняемость | % занятых дней |
| **Rental income** | Доход | Цена × Заполняемость × Дни |
| **Net Profit** | Чистая прибыль | Доход минус 40% |
| **Estimated Price** | Рекомендуемая цена | Net Profit × 10 |

---

## Формула оценки стоимости

```
Estimated Price = Чистая годовая прибыль × 10
```

Стандартный мультипликатор для готовых объектов (10-летняя окупаемость).

### Пример принципа

| Показатель | Значение |
|------------|----------|
| Средняя цена/ночь | $129 |
| Заполняемость | 78% |
| Дней в месяце | 30 |
| Месячная выручка | $3,018 |
| Годовая выручка | $36,216 |
| Чистая прибыль (40%) | $14,486 |
| **Estimated Price** | **$144,860** |

## Интерпретация заполняемости

### Шкала оценки

| Заполняемость | Интерпретация |
|---------------|---------------|
| < 50% | Слабый объект или новый |
| 50-70% | Нормальный уровень |
| 70-85% | Хороший уровень |
| **> 90%** | **Недоиспользованный потенциал** |

### Почему >90% — это плохо

**Пример принципа:**

При заполняемости 97%:
- Можно повысить цену на 20%
- Заполняемость станет 80%
- **Доход вырастет** при меньшей нагрузке

**Проблемы сверхвысокой заполняемости:**
1. Нет времени на обновление
2. Быстрый износ объекта
3. Необходимость закрытия на ремонт
4. Упущенная прибыль от низкой цены

## Анализ сезонности

### Как использовать

1. Выбрать разные месяцы в фильтре
2. Сравнить показатели
3. Понять пики и спады

### Пример принципа: сезонные колебания

| Период | Средняя цена/ночь | Комментарий |
|--------|-------------------|-------------|
| Обычный сезон | $117-120 | Базовая цена |
| Пиковые месяцы | $188-270 | В 1.5-2.2x выше |

### Факторы сезонности

1. **Мусульманские праздники** — резкий скачок
2. **Новогодние праздники** — высокий сезон
3. **Февраль (дожди)** — спад турпотока
4. **Школьные каникулы** — семейный туризм

---

## Ограничения данных

### На что обращать внимание

1. **Available days for rent**
   - Если выбрали 30 дней, а показано 5 — данные неполные
   - Новые номера могут иметь мало истории

2. **Сбои в данных**
   - Редко, но бывают пропуски
   - Много пропусков = недостоверно

3. **Крупные отели**
   - Показывают только 10-15% номеров на Booking
   - Остальное продают напрямую
   - Заполняемость в софте занижена

### Пример принципа: погрешность крупных отелей

| Реальность | В Estate Market |
|------------|-----------------|
| 456 номеров | 50 номеров на Booking |
| Причина | Избегают OTA-комиссий 15% |
| Следствие | Заполняемость кажется ниже |

## Практическое применение

### Для продаж

1. Найти аналог в том же районе
2. Показать клиенту реальные данные через Estate Market
3. Объяснить принцип сравнения

**Принцип диалога:**
- Показать данные по работающим объектам в регионе
- Объяснить, как новый качественный объект может работать лучше старых
- Перенаправить на Zoom для демонстрации конкретных проектов

### Для оценки инвестиций

- Реальная доходность аналогов (исторические данные)
- Сравнение районов по заполняемости
- Понимание паттернов рынка

## Общие паттерны рынка (исторические данные)

> **Важно:** Это общие наблюдения из прошлого опыта. Актуальные данные по конкретным проектам демонстрируются на Zoom-консультации через Estate Market.

### Типичные показатели по классам объектов

На примере прошлых проектов:
- Премиум апартаменты в Чангу обычно показывали заполняемость 70-85%
- Новые качественные объекты часто превосходили старые более дорогие
- Сезонность влияла на цены: пиковые месяцы давали x1.5-2 к базовой цене

### Принцип сравнения объектов

На примере прошлых проектов: качественный апартамент с правильным позиционированием за $100-120k мог приносить сопоставимую или более высокую доходность, чем старый объект за $250-300k.

**Ключевой вывод:** Важнее качество и позиционирование проекта, а не только цена.

### Как объяснять клиенту

1. "На примере прошлых проектов мы видели, что..."
2. "Исторически качественные объекты в этом районе показывали..."
3. "Чтобы показать актуальные данные, предлагаю Zoom — там покажу реальную аналитику"
